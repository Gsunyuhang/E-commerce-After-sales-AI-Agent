模块以 `main.py` 为应用工厂（`create_app()`），通过 `lifespan` 钩子在启动时调用 `database.connection.create_tables` 建表并记录日志；使用 `CORSMiddleware` 开启跨域后，将 `routes/chat.py`、`routes/orders.py`、`routes/eval.py` 三个 `APIRouter` 挂载到 `/api/chat`、`/api/orders`、`/api/eval` 前缀下，并提供 `/health` 与 `/` 两个内置端点。

- 路由层：每个业务域一个文件，仅负责参数解析、调用下游、构造 Pydantic 响应，不持有状态。
- 模型层：`schemas.py` 集中定义所有请求/响应的 Pydantic BaseModel，被各路由共享。
- 依赖方向：路由 → `agent.graph.run_agent`（LangGraph 工作流）、`evaluation.runner.EvaluationRunner`、`database.*`（SQLAlchemy session + models）、`utils.logger`；反向无依赖，保持纯 HTTP 适配层。
- 会话历史与日志落库统一走 `get_db_session()` 上下文管理器，异常捕获后返回 `HTTPException(500)` 或静默记录错误。
