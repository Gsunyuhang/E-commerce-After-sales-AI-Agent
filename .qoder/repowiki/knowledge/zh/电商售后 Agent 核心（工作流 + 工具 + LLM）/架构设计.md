- 三层协作：`agent/graph.py` 用 `StateGraph(AgentState)` 构建 START→intent_recognition→route_by_intent→业务节点(政策答疑/订单查询/退换货/投诉/超范围)→validate→memory_summary→END 的有向图；`tools/*` 实现具体业务能力；`llm/qwen_client.py` 封装 ChatTongyi/DashScopeEmbeddings 供意图识别、LLM 生成与检索使用。
- 工具注册契约：所有工具继承 `tools/base.BaseTool`，实现 `get_parameters_schema` / `_execute` / `validate_params`，并通过 `to_langchain_tool()` 转为 LangChain StructuredTool，由 agent nodes 在运行时调用，执行结果经 `log_tool_call` 写入 `tool_calls`/`tool_results` 并回写 `AgentState`。
- LLM 注入点：`agent/intent.py` 与 `agent/nodes.py` 中的 LLM 调用均通过 `llm.qwen_client.get_qwen_client()` 获取单例，统一从 `config.settings` 读取 dashscope_api_key、模型名与温度参数。
- 状态共享边界：`agent/state.AgentState` TypedDict 是整张图的唯一数据总线，所有 node 只读写该字典键（user_input/session_id/messages/tool_calls/tool_results/workflow_trace/need_human/confidence/slots/context_summary/tur n_count/retrieved_docs/is_retrieval_reliable/retrieval_max_score），不直接跨模块 import 彼此内部函数。
- 单例与入口：`graph.get_agent_graph()` 缓存编译后的图实例；顶层 `run_agent(user_input, session_id, ...)` 构造初始状态、invoke 图并包装为统一返回结构，是上层服务调用的唯一入口。
