模块以 `tools/base.py` 中的抽象基类 `BaseTool` 为核心：子类必须实现 `get_parameters_schema()`（返回 JSON Schema）、`validate_params()`（参数校验）和 `_execute()`（业务逻辑），并由基类的 `execute()` 统一封装校验、日志记录（`utils.logger.log_tool_call`）、异常捕获（将业务异常包装为 `utils.exception.ToolError` / `DatabaseError`）以及标准化返回结构 `{success, data/error, tool_name}`；同时提供 `to_langchain_tool()` 通过 Pydantic `create_model` 动态把 JSON Schema 转为 `langchain_core.tools.StructuredTool`。

四个具体工具均继承自 `BaseTool`，职责单一且互不依赖：
- `order_query.OrderQueryTool`：按 `DD` 前缀正则校验订单号后查 SQLite 的 `Order` 表，并标注售后可用性。
- `logistics_query.LogisticsQueryTool`：先查订单获取 `tracking_number`，再从 `data/seed/logistics.json` 缓存文件匹配物流轨迹。
- `refund_calc.RefundCalcTool`：依据 `return_reason`（`no_reason` / `quality_issue`）与 `has_insurance` 计算商品金额、运费及运费险理赔上限 25 元。
- `ticket_create.TicketCreateTool`：在写入 `Ticket` 前执行资格校验（订单状态、15 天时效、类目黑名单 `NON_RETURNABLE_CATEGORIES` / `NON_EXCHANGEABLE_CATEGORIES`），并去重检查同订单是否存在 `pending` 工单。

依赖方向单向向下：本模块 → `database.connection` + `database.models` + `config.settings` + `utils.*`，不被其他业务模块反向引用，仅作为 Agent 的工具注册源。
