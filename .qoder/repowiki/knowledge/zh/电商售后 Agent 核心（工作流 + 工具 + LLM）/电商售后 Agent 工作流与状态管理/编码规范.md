- 每个 LangGraph 节点函数签名统一为 `(state: AgentState) -> AgentState`，返回新状态字典并通过 `**state` 合并保留未修改字段。
- 外部资源（LLM 客户端、Retriever、Tool、Memory）一律通过模块级 `_xxx: Optional[...] = None` 配合 `get_xxx()` 懒加载单例获取，避免循环导入与启动开销。
- 节点内统一用 `_add_trace(state, "step_name")` 向 `workflow_trace` 追加步骤日志，异常分支也追加 `error=...` 条目以便回溯。
- 需要人工介入的场景统一设置 `need_human=True`，由 `validate_node` 或 `should_validate` 在边界处集中判定是否转人工。
- 对外暴露的常量（如 `INTENT_POLICY_QA` 等）集中在 `intent.py` 顶部定义，并在 `VALID_INTENTS` 白名单中注册，防止散落的字符串字面量。
