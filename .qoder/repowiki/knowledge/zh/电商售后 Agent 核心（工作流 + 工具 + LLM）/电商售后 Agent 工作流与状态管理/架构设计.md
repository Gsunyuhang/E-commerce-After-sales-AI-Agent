模块以 `state.py` 中 `AgentState` TypedDict 为唯一共享状态契约，所有节点通过读写该字典进行信息传递。
- `graph.py`：使用 `langgraph.graph.StateGraph` 构建「感知-规划-执行-校验」有向图，入口为 `intent_recognition`，经 `route_by_intent` 条件路由到 policy_qa / order_query / return_process / complaint / out_of_scope 五个业务节点；policy_qa 与 return_process 需经 `validate` 反幻觉校验后再进入 `memory_summary`，其余直接收尾。提供单例 `get_agent_graph()` 和对外入口 `run_agent()`。
- `nodes.py`：各意图对应的 LangGraph 节点实现，统一签名 `(state: AgentState) -> AgentState`，内部通过延迟初始化的 `_get_*_tool()` 获取订单/物流/工单/退款工具，并追加 `workflow_trace` 用于调试。
- `intent.py`：`IntentRecognizer` 类 + `intent_recognition_node`，调用 Qwen LLM 输出结构化 JSON（intent/confidence/slots/reasoning），低于阈值时置 `need_human=True`。
- `memory.py`：`ConversationMemory` 维护滑动窗口 + 定期 LLM 摘要，按 `memory_summary_interval` 轮次合并旧摘要与新对话片段。
依赖方向：graph → nodes/intent/memory → state；nodes/intent 再向外依赖 `config.prompts/settings`、`llm.qwen_client`、`knowledge.retriever`、`tools.*`、`utils.*`，本模块不反向引用外部层。
