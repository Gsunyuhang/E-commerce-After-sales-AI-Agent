模块由三个职责清晰的子单元组成：
- `runner.py` 提供 `EvaluationRunner` 类作为唯一入口，负责加载 `test_cases.json`、逐条调用 `agent.graph.run_agent` 执行、用 `_evaluate_result` 将期望与真实输出比对（支持 complaint/out_of_scope 宽松匹配、ambiguous 追问订单号特判），并以 `EvalReport`/`EvalResult` 组装报告后持久化到 `data/eval_reports/`。
- `metrics.py` 纯函数式计算层，暴露 `calculate_metrics` 和 `calculate_category_stats`，基于 Pydantic `EvalMetrics` 模型聚合 resolution_rate、refusal_accuracy、avg_response_time、tool_call_success_rate、human_handoff_rate 五项指标。
- `test_cases.json` 是数据驱动的测试集，按 `simple_qa` / `multi_turn` / `ambiguous` / `knowledge_missing` / `out_of_scope` 五类组织，每条用例声明 `expected_intent`、`expected_outcome`、`expected_actions` 等断言字段。
依赖方向单向：`runner` → `metrics` + `agent.graph` + `api.schemas` + `config.settings`，metrics 不反向依赖 runner，便于独立复用。
